## Two options were considered

### Option A — Interactive Logit Simulator ✅ CHOSEN

**What the audience sees:**  
Webpage with three sliders (`cat / fox / dog` logits) and a GT class dropdown. Dragging a slider instantly updates two side-by-side bar charts:

- **Left (navy):** Label Smoothing — downward arrow always on GT class
- **Right (crimson):** MaxSup — downward arrow always on top-1 class

A second section shows what logits look like **after one gradient step**, with a callout: *"LS widens the fox-cat gap → error amplified"* vs *"MaxSup narrows the gap → error corrected."*

**Why this works:**  
The whole paper's thesis is `z_gt → z_max`. The simulator makes that swap *tactile* and directly mirrors the poster's hero figure.

## Implementation — Demo A

### Math (closed-form, no training needed)

```
softmax(z)_k = exp(z_k) / Σ exp(z_j)

∂L_LS/∂z_k     = α(1 - 1/K)  if k == gt
                = -α/K         otherwise

∂L_MaxSup/∂z_k = α(1 - 1/K)  if k == argmax(z)
                = -α/K         otherwise

One gradient step: z_new = z - lr × grad
```

Gradient pushes the target logit **down** and all others slightly **up**.  
In the misclassified case (fox > cat, GT = cat):
- LS step: cat goes down → fox-cat gap widens → **error amplified**
- MaxSup step: fox goes down → fox-cat gap narrows → **error corrected**
